# SPRINT: Home デザイン・実装

## 1. ペルソナ仮説（誰に刺すか）

- **メイン**: Web3プロジェクトのマーケ/イベント担当。AMA・司会・日本展開を検討しており、「信頼できる・実績が分かる・連絡しやすい」を一画面で判断したい。
- **サブ**: 海外プロジェクトの日本コミュニティ担当。日英対応と具体的な数字（フォロワー・イベント数）で即判断したい。
- **仮定**: デザインより「何ができるか・証拠・導線」が優先。画像がなくても文言と構成で伝わることを前提とする。

## 3. A/B/C デザイン方針（短く）

### A: Broadcast Premium
- **色**: 白基調、インク（#0b0f19）の強いコントラスト。アクセントは控えめ。
- **タイポ**: 見出しは太め・大きめ。サブは細めで階層を明確に。
- **余白**: 広め。セクション間の breathing を重視。
- **アニメ**: ほぼなし。ホバーは軽い影・移動のみ。
- **写真**: 人物は「司会・アナウンサー」感。明るく清潔。

### B: Neo Crypto Funk
- **色**: ダーク基調 or 深い紫/青グラデ。アクセント（ネオン・ゴールド）を効かせる。
- **タイポ**: やや遊び（角ばったフォント・大文字キッカー）。
- **余白**: やや詰め。カード・チップを多めに。
- **アニメ**: ホバーで軽いスケール/グラデ変化。オプションで微細なグロー。
- **写真**: イベント・配信の熱量。ネオン/暗部のコントラスト。

### C: Editorial Minimal
- **色**: 白＋グレー。黒に近いテキスト、枠線は細く薄く。
- **タイポ**: 雑誌っぽい。大きな見出し＋小さなリード。行間・字間を広めに。
- **余白**: 最大。写真と文字のブロックで構成。
- **アニメ**: なし。スクロールで「めくる」感を優先。
- **写真**: 大きく使う。1セクション1ビジュアル＋短い文言。

---

## 4. 採用すべき推奨案

**推奨: A 寄り + C の余白（ハイブリッド）**

- **理由**: ペルソナが「信頼・実績・導線」重視のため、Broadcast Premium の清潔さと説得力が合う。C の余白を取り入れることで可読性と「プロ感」を両立。B はキャラが強く、ポートフォリオの第一印象を「遊び」に寄せすぎるリスクがある。
- **実装**: デフォルトを A ベースにし、余白・フォントサイズは C を参考にやや広めに設定。B は __design での比較用に残す。

## 5. まだ不明な点（最大3）

1. **写真の差し替えタイミング**: ヒーロー・Gallery の実写真をいつ投入するか（CMS 優先か、ローカル一括か）。現状はプレースホルダー/既存画像で成立させる。
2. **本番のテーマ固定**: __design で A/B/C を試したあと、本番は「A寄り+C余白」で固定するか、ユーザー切替を残すか。現時点では固定を推奨。
3. **英語フォント**: 日本語は Noto Sans JP のまま。英語のみ別フォント（例: Inter）を入れるかは未決定。後から追加可能な構造にはしてある。

## 6. コピー案（3トーン）— Copywriter

### Hero 見出し
| トーン | 日本語 | 用途 |
|--------|--------|------|
| 硬め | Web3イベント司会・AMAホスト（日英）。課題を「伝わる」と「動く」に変えます。 | 法人・公式向け |
| 標準 | Web3を「分かる」から「動く」へ。日英MC・AMAホストのAki | 現行に近い |
| 軽め | 難しい話、短く・具体に。日英でMC/AMAやります。 | SNS寄り |

### Hero サブ
| トーン | 日本語 |
|--------|--------|
| 硬め | 議題設計・質問設計・テンポ設計により、理解と参加を両立するセッションを提供します。 |
| 標準 | 難しい話ほど、短く・具体に。AMAなら「質問設計→テンポ→要点回収」で、コメントが動く流れを作ります。 |
| 軽め | 盛り上げつつ、置いていかない進行。まずはDMで相談どうぞ。 |

### 主CTA
| トーン | 主 | 副 |
|--------|-----|-----|
| 硬め | 実績一覧を見る | サービス内容を見る |
| 標準 | 実績を見る | サービスを見る |
| 軽め | 実績チェック | サービスチェック |

採用: 標準トーンをデフォルト。差し替えは `copy-pack` または CMS で可能。

## 7. 実装メモ

- デザイントークン: `lib/theme.ts` と `app/globals.css` の CSS 変数＋`data-theme`。
