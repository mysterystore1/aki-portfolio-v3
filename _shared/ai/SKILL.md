# SKILL (Workspace Rules)

## 何をする場所？
- C:\work は複数プロジェクトを“チーム扱い”で運用する作業場。
- 共通ルールは _shared/ai に集約し、各プロジェクトは差分だけ扱う。

## 参照順（最重要）
1. ./_shared/ai/SKILL.md
2. ./_shared/ai/PROJECTS.md の「該当プロジェクト(フォルダ名)」セクション
3. そのプロジェクトの README / package.json / 設定ファイル

## スコープ
- 触って良い範囲：C:\work 配下のみ（projects と _shared）
- ワークスペース外（C直下、Users直下、他ドライブ）へ勝手にアクセス/変更しない

## セキュリティ
- APIキー/秘密鍵/トークン/個人情報は出力に含めない・ログに出さない
- .env / secrets は参照しない・貼らない
- 破壊的操作（大量削除/大量置換/大規模リネーム）は事前に確認する

## 作業の進め方
- まず結論→理由→次の手順で提案する
- 不明点は最大3つだけ質問し、止まらず仮定で前進（仮定は明記）
- 変更は最小差分（関係ない整形や大改修は避ける）
- 可能なら lint/test/build を回してから提案する（PROJECTS.mdのコマンドに従う）

## 出力
- コマンドはコピペできる形で提示
- 変更があるなら「対象ファイル」と「変更点」を明示する
