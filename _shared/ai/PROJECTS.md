- 目的：デザイン改善・機能追加・不具合修正を、最小差分で継続的に進める。優先順は「不具合修正 -> デザイン改善 -> 機能追加」。
- 実行：
  - `pnpm install`
  - `pnpm lint`
  - `pnpm build`
  - `pnpm start`
  - スモーク確認: `/jp` 表示、`/en` フォールバック表示、`/works/[slug]` の画像比率
- 触るな：
  - `_shared/**`（共通ルール領域）
  - `.env*` と秘密情報（APIキー/トークン/個人情報）
  - microCMS の既存フィールド名・API契約（互換性を崩す変更）
